# მთავარი გვერდის სექციების მართვა

ეს სისტემა საშუალებას გაძლევთ ადმინ პანელიდან მართოთ მობილური აპლიკაციის მთავარი გვერდის სექციები.

## 🚀 დაყენება

### 1. Backend - სექციების ბაზაში ჩაწერა

პირველად უნდა შეასრულოთ seed სკრიპტი, რომ შექმნას ნაგულისხმევი სექციები:

```bash
cd aphoteka-backend
npx ts-node src/seed/seed-home-sections.ts
```

ეს ჩაწერს 4 ნაგულისხმევ სექციას:
1. **მედიკამენტი** (category) - ყველა მედიკამენტი
2. **კოსმეტიკა** (category) - კოსმეტიკის კატეგორია
3. **შენი რჩეული** (favorites) - მომხმარებლის რჩეული პროდუქტები
4. **ფასდაკლებული** (discounted) - ფასდაკლებული პროდუქტები

### 2. Backend - სერვერის გაშვება

დარწმუნდით რომ backend სერვერი გაშვებულია:

```bash
cd aphoteka-backend
npm run start:dev
```

## 📱 ადმინ პანელი

### სექციების მართვა

1. გახსენით ადმინ პანელი
2. მენიუში აირჩიეთ: **მობილური აპლიკაცია → მთავარი გვერდის სექციები**
3. აქ შეგიძლიათ:
   - ✅ **დაამატოთ ახალი სექციები** - "+ ახალი სექცია" ღილაკი
   - ✏️ **რედაქტირება** - არსებული სექციების შეცვლა
   - 👁️ **ხილვადობის გადართვა** - სექციის დამალვა/გამოჩენა
   - ⬆️⬇️ **რიგითობის შეცვლა** - ისრებით სექციების გადალაგება
   - ❌ **წაშლა** - სექციის წაშლა

### სექციის ტიპები

#### 1. **კატეგორია** (category)
- გამოჩნდება კონკრეტული კატეგორიის პროდუქტები
- შეგიძლიათ აირჩიოთ კატეგორია dropdown-დან
- თუ კატეგორია არ აირჩევთ - გამოჩნდება ყველა მედიკამენტი (კოსმეტიკის გარეშე)

#### 2. **ფასდაკლებული** (discounted)
- ავტომატურად გამოჩნდება ყველა ფასდაკლებული პროდუქტი

#### 3. **რჩეული** (favorites)
- გამოჩნდება მომხმარებლის რჩეული (favorites) პროდუქტები
- თუ favorites ცარიელია - გამოჩნდება შესაბამისი შეტყობინება

#### 4. **ყველა** (all)
- გამოჩნდება ყველა პროდუქტი (მედიკამენტი + კოსმეტიკა)

### სექციის პარამეტრები

- **სახელი** - სექციის სათაური (მაგ: "მედიკამენტი", "ახალი პროდუქტები")
- **ტიპი** - სექციის ტიპი (იხ. ზემოთ)
- **კატეგორია** - (მხოლოდ category ტიპისთვის) რომელი კატეგორიის პროდუქტები გამოჩნდეს
- **ლიმიტი** - მაქსიმალური პროდუქტების რაოდენობა (1-50)
- **ხილვადი** - სექცია გამოჩნდეს თუ არა მობილურ აპლიკაციაში

## 📱 მობილური აპლიკაცია

მობილური აპლიკაცია ავტომატურად იტვირთება სექციებს API-დან და აჩვენებს მათ მთავარ გვერდზე.

### როგორ მუშაობს:

1. აპლიკაცია იტვირთება მთავარ გვერდზე
2. ხდება `/home-sections/visible` API-ის გამოძახება
3. მიიღება მხოლოდ ხილვადი სექციები, რიგითობის მიხედვით დალაგებული
4. თითოეული სექცია იტვირთება შესაბამისი პროდუქტებით
5. სექციები გამოჩნდება Horizontal ScrollView-ში

## 🔄 API Endpoints

### Backend Endpoints:

```
GET    /home-sections          - ყველა სექცია (admin-სთვის)
GET    /home-sections/visible  - მხოლოდ ხილვადი სექციები (mobile app-სთვის)
GET    /home-sections/:id      - კონკრეტული სექცია
POST   /home-sections          - ახალი სექციის შექმნა (auth required)
PATCH  /home-sections/:id      - სექციის განახლება (auth required)
DELETE /home-sections/:id      - სექციის წაშლა (auth required)
POST   /home-sections/reorder  - სექციების რიგითობის შეცვლა (auth required)
```
