# კატეგორიების Seeding

## მიმოხილვა

ეს სკრიპტი ავტომატურად ქმნის ყველა საჭირო კატეგორიას სურათებით MongoDB-ში.

## კატეგორიები

სკრიპტი ქმნის 13 კატეგორიას:

1. **დედა და ბავშვი** - საბავშვო საჭიროებები, ბავშვის მოვლა
2. **კოსმეტიკა და პირადი ჰიგიენა** - პირადი ჰიგიენის საშუალებები
3. **პირის ღრუს მოვლა** - კბილის ჯაგრისები, პასტები, ღრძილებები
4. **დეკორატიული კოსმეტიკა** - მაკიაჟის პროდუქტები
5. **მამაკაცის ჰიგიენა** - მამაკაცის მოვლის საშუალებები
6. **მედიკამენტები** - სამედიცინო პრეპარატები
7. **სტატიკური მოწყობილობა** - სამედიცინო აღჭურვილობა
8. **სამედიცინო მოწყობილობა** - დიაგნოსტიკური მოწყობილობები
9. **სასმელები და სუპლემენტები** - ვიტამინები და დანამატები
10. **სავარძლები და ბამბის ფუფხები** - საწოლები და სავარძლები
11. **საცდელი დანიშნულების ნაწარმი** - ტესტები და საცდელი აღჭურვილობა
12. **ვიტამინები** - ვიტამინები და მინერალები
13. **ანტისეპტიკები** - დეზინფექციის საშუალებები

## როგორ გამოვიყენოთ

### 1. კატეგორიების შექმნა

```bash
cd aphoteka-backend
npm run seed:categories
```

### 2. რას აკეთებს სკრიპტი?

- ✅ ამოწმებს არსებობს თუ არა კატეგორიები
- ✅ თუ არ არსებობს - ქმნის ახალ კატეგორიებს
- ✅ თუ არსებობს - მხოლოდ განაახლებს სურათებს და ფერებს
- ✅ თითოეულ კატეგორიას ამატებს:
  - სახელი (name)
  - აღწერა (description)
  - სურათის URL (imageUrl) - Unsplash-დან
  - ფერი (color)
  - აიქონი (icon)
  - sortOrder

### 3. უსაფრთხოება

სკრიპტი უსაფრთხოა:
- არ წაშლის არსებულ კატეგორიებს
- არ ცვლის კატეგორიის სახელებს
- თუ კატეგორია უკვე არსებობს, მხოლოდ განაახლებს სურათს და მეტადატას

## Troubleshooting

### MongoDB Connection Error
```bash
Error: querySrv ECONNREFUSED
```
**გადაწყვეტა**: დარწმუნდი რომ MongoDB URI სწორია `.env` ფაილში

### Permission Error
```bash
Error: Cannot create category
```
**გადაწყვეტა**: დარწმუნდი რომ MongoDB მომხმარებელს აქვს write permissions
